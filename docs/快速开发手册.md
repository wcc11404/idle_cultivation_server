## 默认开发规范

- 文件夹使用小写加下划线命名。
- Python 源码文件默认使用首字母大写命名。
- 类名使用首字母大写。
- 函数、局部变量、实例变量使用小写加下划线。
- 常量使用大写加下划线。
- 以下 Python 约定入口保留现名，不强制改成首字母大写：
  `__init__.py`、`conftest.py`、`test_*.py`、`main.py`、`integration_test.py`
- 前导下划线函数/变量视为私有的小写下划线写法，属于合规形式。
- 接口中文提示不在服务端拼接，业务语义统一通过 `reason_code + reason_data` 输出。
- 修改接口行为时，优先同步 `api_documentation.md`；修改实现约定时，再同步对应模块文档。
- 每日重置判定真值是 `player_data.last_daily_reset_at`，不是 `last_online_at`。
- 每日重置触发点是“登录 + 任意写请求公共入口（`get_write_game_context`）”，确保玩家跨4点在线也会重置。

## 按任务查文档

- 改接口字段、`reason_code`、返回结构：
  先看 `api_documentation.md`，再看对应模块文档。
- 改模块逻辑、互斥、状态流：
  直接看对应 `模块-*.md`。
- 改部署脚本、服务器环境、启动问题：
  看 `安装与启动手册.md`。
- 线上故障排查、数据库重建、字段缺失修复：
  看 `线上运维手册.md`。
- 改认证、防作弊、锁、敏感词：
  先看 `模块-游戏基础与公共能力.md`，安全专项看 `security_system.md`。
