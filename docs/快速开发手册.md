## 默认开发规范

- 文件夹使用小写加下划线命名。
- Python 源码文件默认使用首字母大写命名。
- 类名使用首字母大写。
- 函数、局部变量、实例变量使用小写加下划线。
- 常量使用大写加下划线。
- 以下 Python 约定入口保留现名，不强制改成首字母大写：
  `__init__.py`、`conftest.py`、`test_*.py`、`main.py`、`integration_test.py`
- 前导下划线函数/变量视为私有的小写下划线写法，属于合规形式。
- 接口中文提示不在服务端拼接，业务语义统一通过 `reason_code + reason_data` 输出。
- 修改接口行为时，优先同步 `api_documentation.md`；修改实现约定时，再同步对应模块文档。
- 每日重置判定真值是 `player_data.last_daily_reset_at`，不是 `last_online_at`。
- 每日重置触发点是“登录 + 任意写请求公共入口（`get_write_game_context`）”，确保玩家跨4点在线也会重置。

## SQL 初始化约定

- 数据库首轮初始化入口仍是 `sql/init.sql`。
- `sql/init.sql` 现在只负责按顺序加载 `sql/schema/*.sql`。
- 不同子系统必须拆成独立 SQL 文件维护，不能继续把所有建表语句堆回单文件。
- 重要字段必须在 SQL 中直接写注释，至少覆盖：
  - 主键/外键真值
  - 状态位
  - 软删字段
  - token / confirm token
  - JSON 快照字段

推荐重建命令：

```bash
bash sql/init_db.sh
```

## 测试账号并发约束

- 同一测试账号不允许被多个 pytest 进程并发使用。
- 原因不是“工具限制”，而是服务端本身就有单账号踢下线机制；同账号并发登录会触发 `KICKED_OUT`，并污染任务、修炼、炼丹、采集等累计型测试结果。
- `unit_test/conftest.py` 已加入会话级测试账号占用锁：
  - 默认账号名仍会按进程生成 `test_<pid>`
  - 如果手动设置 `IDLE_TEST_USERNAME`，请确保不同并发进程使用不同值
  - 若同一账号已被其他 pytest 进程占用，测试会在启动阶段直接失败并给出明确提示
