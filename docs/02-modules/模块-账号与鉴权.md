# 模块-账号与鉴权

## 职责与边界

负责注册、登录、刷新 token、登出、改密码、改昵称、改头像，以及登录/登出时的运行态重置。

## 入口接口

- `/api/auth/register`
- `/api/auth/login`
- `/api/auth/refresh`
- `/api/auth/logout`
- `/api/auth/change_password`
- `/api/auth/change_nickname`
- `/api/auth/change_avatar`

## 关键数据与配置

- `accounts` 表账号信息
- token 版本号与 JWT
- 登录/登出时统一清理修炼、采集等临时运行态

## 主要状态流转

- 注册成功后直接返回 token
- 登录成功后返回账号信息与整包游戏数据
- refresh 只负责续签，不负责业务提示
- 登出与登录共用运行态重置逻辑，避免遗留临时状态跨会话残留

## 互斥与依赖

- 依赖 `Security`、`Dependencies`、`InitPlayerInfo`
- 昵称合法性依赖敏感词和字符校验

## reason_code 语义出口

以 `ACCOUNT_*` 为主，包括注册、登录、昵称、头像修改成功与失败语义。

## 测试覆盖点

- `unit_test/test_auth_game_api.py`
- `unit_test/test_edge_cases_api.py`
- `unit_test/integration_test.py`

## 维护注意事项

- 登录成功返回的数据结构必须和客户端启动链路保持一致
- 运行态重置逻辑改动时，要同步检查修炼、采集、炼丹、历练状态清理
