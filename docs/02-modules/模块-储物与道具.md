# 模块-储物与道具

## 职责与边界

负责背包容量、物品使用、丢弃、整理、扩容，以及解锁术法/丹方/丹炉类道具效果。

## 入口接口

- `/api/game/inventory/list`
- `/api/game/inventory/use`
- `/api/game/inventory/discard`
- `/api/game/inventory/expand`
- `/api/game/inventory/organize`

## 关键数据与配置

- `items.json`
- 背包 `slots` 与 `capacity`
- 使用结果中的 `effect`、`contents`

## 主要状态流转

- use 按物品类型走消耗品、礼包、术法书、丹方、丹炉等分支
- use 支持 `count`（接口不变）：
  - 单次请求内部循环执行
  - 返回 `requested_count/completed_count/is_partial`
  - 全部成功返回原成功码；部分成功返回 `INVENTORY_USE_PARTIAL_SUCCEEDED`
- discard 扣减指定物品数量
- expand 提升容量上限
- organize 重排格子但不改变总量

## 互斥与依赖

- 术法书/丹方/丹炉会联动术法系统和炼丹系统
- 关键提示统一由客户端根据 `reason_code + reason_data` 翻译

## reason_code 语义出口

以 `INVENTORY_*` 为主，包括使用成功、重复使用、数量不足、不可用、扩容成功等。
批量场景额外包含：`INVENTORY_USE_PARTIAL_SUCCEEDED`（客户端需按 completed/requested 展示）。

## 维护注意事项

- `use` 返回字段是客户端文案的结构化输入，改字段前必须同步 API 手册与客户端映射
- 若调整 use 的批量汇总字段（`requested_count/completed_count/is_partial`），需同步：
  - 客户端储纳模块文案与刷新逻辑
  - `unit_test/test_cultivation_inventory_api.py` 对批量路径断言
