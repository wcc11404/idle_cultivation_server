# 模块-修炼与突破

## 职责与边界

负责修炼开始、秒制上报、停止修炼、突破判定与结算，以及境界静态配置读取。

## 入口接口

- `/api/game/player/cultivation/start`
- `/api/game/player/cultivation/report`
- `/api/game/player/cultivation/stop`
- `/api/game/player/breakthrough`

## 关键数据与配置

- `realms.json`
- 玩家 `is_cultivating`
- `last_cultivation_report_time`
- `cultivation_effect_carry_seconds`

## 主要状态流转

- start 进入修炼态
- report 按小数秒上报，但只按整秒结算修炼收益
- carry 余量写回玩家数据，供下一次 report 继续结算
- breakthrough 读取当前灵气、灵石与材料后做突破判定，不负责客户端提示拼接

## 互斥与依赖

- 与炼丹、历练、草药采集互斥
- 依赖 `PlayerSystem`、`RealmData`、`SpellSystem`

## reason_code 语义出口

以 `CULTIVATION_*` 为主，覆盖开始、上报、停止、突破成功失败和时间非法。

## 测试覆盖点

## 维护注意事项

- 秒制上报和整秒结算必须同步保持
- 反作弊校验看上报秒数与真实时间差，不要和客户端乐观显示混淆
