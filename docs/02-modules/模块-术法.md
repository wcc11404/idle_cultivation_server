# 模块-术法

## 职责与边界

负责术法解锁、列表读取、装备/卸下、升级、充灵、升星、属性加成汇总，以及主动/开局术法在战斗内的实际生效。

## 入口接口

- `/api/game/spell/list`
- `/api/game/spell/equip`
- `/api/game/spell/unequip`
- `/api/game/spell/upgrade`
- `/api/game/spell/charge`
- `/api/game/spell/star_up`

## 本轮重要变更总览

### 1. 配置结构升级

- `spells.json` 已升级为新结构：
  - `type`
  - `rarity`
  - `element`
  - `max_level`
  - `max_star`
  - `unlock_item_id`
  - `levels`
  - `stars`
- `effect` 统一为 `List[Dictionary]`。
- 每条 effect 使用 `effect_type` 区分。
- 旧术法 `thunder_strike`（雷击术）已删除。
- 客户端与服务端术法真值已保持同步。

### 2. 术法分类与槽位

- 吐纳：`breathing`
- 主动：`active`
- 开局：`opening`
- 生产：`production`
- 槽位限制：
  - `breathing = 1`
  - `active = 2`
  - `opening = 2`

### 3. 升级与升星

- 升级依赖：
  - 使用次数
  - 已充灵气
- 升星依赖：
  - 同名术法解锁道具
  - `blank_jade_slip`（空白玉简）
- 升星口径：
  - 升几星就需要几个对应术法解锁道具
  - 从 `2 -> 3` 开始，每次固定需要 `1` 个空白玉简
- `star_up_spell(...)` 成功后：
  - 扣除道具
  - 增加星级
  - 重新计算术法加成缓存

### 4. 属性加成口径

当前术法属性分两类：

- 乘法类：
  - `attack`
  - `defense`
  - `health`
  - `max_spirit`
  - `spirit_gain`
  - `penetration`
  - `crit_damage`
- 加法类：
  - `speed`
  - `hit`
  - `dodge`
  - `crit`
  - `anti_crit`

服务端 `SpellSystem` 会先汇总单个术法的等级与星级加成，再生成：
- 乘法类总倍率
- 加法类总值

随后由 `AttributeCalculator` 统一参与静态属性和动态属性计算。

### 5. 开局术法统一口径

- `attack / defense / penetration / crit_damage / health` 使用 `buff_percent`。
- `hit / dodge / crit / anti_crit` 也统一使用 `buff_percent`，不再与 `buff_value` 混用。
- `speed` 保持使用 `buff_value`。
- 双属性开局术法每级成长按绝对值增长，例如：
  - `12.5% -> 13.75%`
  - 不是乘法放大。

### 6. 主动术法与开局术法战斗生效

- 主动术法 effect 支持：
  - `instant_damage`
  - `drain_health`
  - `turn_gauge_delta`
- 开局术法 effect 统一使用：
  - `undispellable_buff`
- 开局术法会在战斗开始时直接注入当前战斗属性，作为常驻 Buff。

## 关键数据与配置

- `app/game/content/spell/spells.json`
- 玩家 `player_spells`
- 已装备槽位 `equipped_spells`
- 当前缓存：术法属性加成 `bonuses`

## 主要状态流转

- 解锁道具使用后，术法进入玩家 `player_spells`
- `equip/unequip` 修改装备槽位
- `upgrade` 消耗使用次数和充灵
- `charge` 扣减玩家灵气并增加 `charged_spirit`
- `star_up` 扣减解锁道具与空白玉简后增加星级
- 每次状态变化后重新计算术法属性缓存

## 关键函数位置

### 配置与缓存
- `app/game/domain/spell/SpellData.py`
  - 术法配置加载与读取
- `app/game/domain/spell/SpellSystem.py`
  - 玩家术法存档、装备槽位、升级/充灵/升星、术法属性加成汇总

### 静态与动态属性
- `app/game/domain/player/AttributeCalculator.py`
  - 静态属性计算
  - 动态属性计算
  - 命中/暴击/伤害统一结算
- `app/game/domain/player/PlayerSystem.py`
  - 静态属性写回

### 战斗流程
- `app/game/domain/lianli/LianliSystem.py`
  - 历练战斗主循环
  - 玩家/敌人普攻
  - 开局术法注入

## reason_code 语义出口

以 `SPELL_*` 为主，覆盖：
- 槽位上限
- 战斗锁定
- 升级条件不足
- 充灵成功/失败
- 升星成功/失败

## 维护注意事项

- 术法属性加成是服务端真值，客户端只做展示与静态属性计算辅助。
- 修改 `spells.json` 后，若运行中表现不对，优先确认：
  - 服务端 `SpellData` 配置是否已重新加载
  - `spell/list` 返回的 `spells_config` 是否为新值
- 开局术法与主动术法都以 `effects[]` 为真值，不建议再让固定 description 承担唯一业务语义。
