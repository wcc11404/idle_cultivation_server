# 模块-历练

## 职责与边界

负责普通区域、每日区域、试炼塔战斗模拟与结算，以及区域/敌人配置读取。

## 入口接口

- `/api/game/lianli/speed_options`
- `/api/game/lianli/simulate`
- `/api/game/lianli/finish`
- `/api/game/dungeon/foundation_herb_cave`
- `/api/game/tower/highest_floor`

## 关键数据与配置

- `areas.json`
- `enemies.json`
- 当前战斗数据、塔层、每日副本次数

## 主要状态流转

- `speed_options` 根据当前境界与 VIP 状态动态返回可用倍速集合
- simulate 生成战斗时间轴、敌人数据、总时长与预期掉落
- finish 先校验 `speed` 合法性，再按 `speed + index` 校验时间并结算
- 塔层和每日副本信息通过查询接口单独读取

## 互斥与依赖

- 与修炼、炼丹、草药采集互斥
- 结算会联动玩家血量、背包掉落、塔层推进

## reason_code 语义出口

以 `LIANLI_*` 为主，覆盖倍速选项、模拟成功、时间非法、倍速非法、部分结算、全部结算、副本次数不足等。

## 测试覆盖点

- `unit_test/test_spell_alchemy_lianli_api.py`
- `unit_test/test_edge_cases_api.py`
- `unit_test/integration_test.py`

## 维护注意事项

- `speed_options` 与 `finish.speed` 的合法性判定必须保持同一套规则
- `finish` 的 `index`、`speed` 和时间校验是客户端联动敏感点，改动前必须同步测试
