# 模块-任务系统

## 职责与边界

负责每日任务/新手任务配置读取、进度推进、领奖判定与发奖。任务进度和奖励发放均由服务端权威控制。

## 入口接口

- `/api/game/task/list`
- `/api/game/task/claim`

## 关键数据与配置

- `app/modules/task/tasks.json`
- `TaskData`（配置读取）
- `TaskSystem`（任务状态、进度封顶、领奖逻辑）
- 玩家 `task_system`：
  - `daily_tasks.{task_id}.progress / claimed`
  - `newbie_tasks.{task_id}.progress / claimed`

## 主要状态流转

- `task/list`：返回每日与新手任务全量状态（含 progress/target/completed/claimed/rewards）。
- `task/claim`：校验任务存在、是否完成、是否已领取，通过后发奖并标记 `claimed=true`。
- 任务进度到 `target` 后封顶，不再增长。

## 互斥与依赖

- 任务系统本身无运行态互斥。
- 进度由其他模块在“成功分支”挂点推进：
  - 修炼 report（秒数）
  - 历练 finish（按次）
  - 炼丹 report（按请求成功次数）
  - 采集 report（按请求成功次数）
  - 术法 charge（按请求成功次数）
  - 背包 use（新手礼包Ⅰ/Ⅱ/Ⅲ）

## reason_code 语义出口

- `TASK_LIST_SUCCEEDED`
- `TASK_CLAIM_SUCCEEDED`
- `TASK_CLAIM_TASK_NOT_FOUND`
- `TASK_CLAIM_NOT_COMPLETED`
- `TASK_CLAIM_ALREADY_CLAIMED`

## 重置约定

- 每日任务：参与每日重置，跨天后回到“未完成/未领取”。
- 新手任务：一次性任务，不参与每日重置。
- 登录/登出运行态重置与任务系统解耦；任务系统不持有运行中状态。
