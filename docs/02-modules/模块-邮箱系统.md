# 模块-邮箱系统

## 职责与边界

- 提供玩家邮箱能力：列表、详情（含已读标记）、附件领取、删除。
- 邮件真值存储在独立表 `mail_data`，不放入 `player_data.data`。
- 删除采用软删（`is_deleted=true`），默认查询过滤软删数据。

## 关键数据结构

- `mail_id/account_id/title/content/attachments`
- `created_at/expire_at`
- `is_read/is_claimed/claimed_at`
- `is_deleted/deleted_at/updated_at`

## 核心规则

- 单账号邮箱容量上限 100，超限拒绝新邮件写入。
- 附件最多 10 种物品类型（同 `item_id` 去重）。
- 详情接口会标记已读。
- 删除约束：
  - 手动删除：有附件且未领取时禁止删除；无附件或附件已领取允许删除。
  - 一键删除：删除“已读且非未领取”邮件（已读无附件 + 已读已领取）。
- 领取规则：
  - 附件整封领取。
  - 任一附件入包失败即整体回滚，不产生脏写。

## 对外接口

### 游戏接口

- `GET /api/game/mail/list`
- `GET /api/game/mail/detail`
- `POST /api/game/mail/claim`
- `POST /api/game/mail/delete`

### 管理接口

- `POST /api/admin/mail/send`
- `POST /api/admin/mail/send_batch`

## 维护建议

- 批量运营发信优先走 `admin/mail/send_batch`，避免手工 SQL 修改。
- 清理历史邮件时保持软删语义，不直接物理删除。
