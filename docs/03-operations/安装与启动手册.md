# 安装与启动手册

这份手册覆盖本地开发、Ubuntu/腾讯云部署，以及现有启动脚本的使用方式。

## 最短流程

### 首次部署（Ubuntu / 腾讯云）

```bash
cd idle_cultivation_server
bash setup_ubuntu.sh
bash restart.sh
```

### 日常重启

```bash
cd idle_cultivation_server
bash restart.sh
tail -f server.log
```

## setup_ubuntu.sh

用途：
- 安装 Python、PostgreSQL、编译工具和常用命令
- 创建或修复 `venv`
- 安装 `requirements.txt`
- 初始化数据库
- 可选在安装后直接启动服务

支持参数：
- `--start`：安装后直接执行 `restart.sh`
- `--skip-apt`：跳过 apt 安装
- `--recreate-venv`：强制删除并重建 `venv`

默认行为：
- 会自动检测 `venv/bin/python` 是否可用
- 检测到跨机器拷贝导致的坏 `venv` 时，会自动重建
- 会确保数据库 `idle_cultivation_game` 和 `pgcrypto` 扩展存在

## restart.sh

用途：
- 检查 `venv/bin/python` 是否存在
- 检查并启动 PostgreSQL
- 清理占用 `8444` 端口的旧 uvicorn 进程
- 以 `venv/bin/python -m uvicorn app.main:app --reload` 启动服务

关键说明：
- 日志写到根目录 `server.log`
- 默认监听 `0.0.0.0:8444`
- `--reload` 仅适合开发环境；生产部署如果后续需要 systemd，可再单独拆

## stop.sh

用途：
- 停止当前 uvicorn 进程
- 停止 PostgreSQL 服务

当前更适合本地开发或单机测试环境；线上如果后续改成常驻进程管理，停止方式也应一并调整。

## 本地开发环境

```bash
cd idle_cultivation_server
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
bash restart.sh
```

数据库默认使用本机 PostgreSQL：
- macOS：`postgres://hsams:hsams@localhost:5432/idle_cultivation_game`
- Linux：`postgres://postgres:postgres@localhost:5432/idle_cultivation_game`

## 常用排查

- 服务没起来：
  先看 `server.log`
- `venv/bin/python` 不可执行：
  直接 `bash setup_ubuntu.sh --recreate-venv`
- 8444 端口冲突：
  `restart.sh` 会自动清理；如果仍异常，手动检查 `lsof -i:8444`
- 数据库初始化异常：
  先确认 PostgreSQL 服务状态，再检查 `sql/init.sql` 是否已经执行过

## 测试

```bash
pytest unit_test
python -m unit_test.integration_test
```

测试默认依赖服务和数据库可用，基址默认是 `http://localhost:8444/api`。
