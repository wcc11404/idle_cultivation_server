# 运维系统安装与启动手册

## 1. 适用范围

这份手册用于说明当前线上可视化运维系统的一期环境要求、安装步骤、启动方式和基本排障。

当前运维系统由两部分组成：

- 服务端：`app/ops/*`
- 前端：`ops_web/*`

当前一期已落地的主要页面：

- 登录页
- 系统概览页
- 玩家管理页
- 发放中心
- 审计日志页
- 系统维护页

---

## 2. 环境要求

### 2.1 基础服务端环境

必须具备：

- `Python 3`
- `venv`
- `pip`
- `PostgreSQL`

### 2.2 运维前端环境

必须具备：

- `Node.js`
- `npm`

### 2.3 当前推荐方式

服务端基础环境继续沿用：

- `setup_ubuntu.sh`
- `docs/03-operations/安装与启动手册.md`

运维系统属于在现有服务端上的增量能力，不单独另起一套部署栈。

---

## 3. 安装步骤

### 3.1 只安装服务端基础环境

```bash
cd idle_cultivation_server
bash setup_ubuntu.sh
```

这会完成：

- Python / venv / pip
- PostgreSQL
- 主数据库初始化
- 服务端依赖安装

### 3.2 安装包含运维系统的环境

```bash
cd idle_cultivation_server
bash setup_ubuntu.sh --with-ops-web --init-ops-db
```

这会额外完成：

- 安装 `Node.js` 与 `npm`
- 初始化 `ops_*` 相关表与引导账号
- 安装 `ops_web` 前端依赖
- 构建 `ops_web/dist`

如果只想装前端依赖但暂时不构建：

```bash
bash setup_ubuntu.sh --with-ops-web --skip-ops-web-build
```

## 4. 本地开发方式

### 4.1 启动服务端

```bash
cd idle_cultivation_server
bash restart.sh
```

默认地址：

- 游戏接口文档：`http://localhost:8444/api/docs`
- 运维网页：`http://localhost:8444/ops`
- 运维接口：`http://localhost:8444/ops/api/*`

补充说明：

- `restart.sh` 当前走的是本地热更启动链路。
- 如果你遇到浏览器偶发 `502 Bad Gateway`，优先判断是否是热更链路短暂抖动，而不是先怀疑 `ops_web` 构建失败。
- 遇到这类情况，可直接用一条干净的临时命令验证页面本身是否正常：

```bash
cd idle_cultivation_server
./venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8445
```

然后访问：

- `http://127.0.0.1:8445/ops`

### 4.2 启动运维前端开发服务器

```bash
cd idle_cultivation_server/ops_web
npm install
npm run dev
```

默认前端开发地址：

- `http://localhost:5173`

开发期代理：

- `Vite` 会将 `/ops/api/*` 代理到 `http://127.0.0.1:8444`

前端构建注意事项：

- `ops_web` 生产构建依赖：
  - `vite.config.ts` 中的 `base: '/ops/'`
  - `BrowserRouter basename='/ops'`
- 如果忘了其中任意一项，典型现象会是：
  - `/ops` 能返回 HTML
  - 但静态资源 404，或者页面白屏

当前页面交互约定：

- 玩家管理：
  - 展示用户名、昵称、注册时间、境界、封禁状态
  - 支持按用户名或昵称搜索
  - 时间统一按东八区显示
- 发放中心：
  - 只允许发送带附件邮件
  - 一封邮件支持多个附件
  - 每个附件通过下拉框按物品中文名选择
  - 不允许直接把物品发进玩家背包
- 审计日志：
  - 时间统一按东八区显示
- 系统维护：
  - 支持登录闸门
  - 支持白名单
  - 支持全部踢下线

## 5. 运维引导账号

当前一期默认会由启动流程保证引导运维账号存在。

默认配置来自：

- `app/core/config/ServerConfig.py`

当前默认值：

- 用户名：`admin`
- 密码：`admin123`

说明：

- 这不是长期推荐的生产密码方案
- 正式环境应尽快改为受控配置或初始化后立即修改
- 默认测试账号 `test`、`test2` 会自动进入登录闸门白名单
  - 新库初始化时会写入
  - 应用启动时也会幂等补齐

## 6. 排障建议

### 6.1 打开 `/ops` 显示“ops_web 未构建”

说明前端产物还没生成。

执行：

```bash
cd idle_cultivation_server/ops_web
npm install
npm run build
```

### 6.2 `npm install` 权限报错

如果遇到本机缓存权限问题，可临时改用本地缓存目录：

```bash
npm_config_cache=.npm-cache npm install
```

### 6.3 `/ops/api/*` 返回 401

先检查：

- 是否已登录并拿到运维 token
- `ops_users` 中账号是否存在且启用
- `ops_auth_tokens` 是否已被撤销或过期

### 6.4 普通玩家无法登录

先检查：

- `ops_system_state.login_gate_enabled` 是否开启
- `ops_login_whitelist` 是否只放行了白名单账号
- 玩家账号本身是否已被永久封禁

### 6.5 白名单添加时报账号不存在

当前白名单接口支持两种输入：

- 账号 UUID
- 用户名

如果添加失败，优先检查：

- 用户名是否输入正确
- 该账号是否已经注册
- 是否误把昵称当成用户名输入
